# User Use Cases

As a DM, I want to create, manage.

As a user, I want to store my project files remotely.

As a user, I want to be able to undo, and redo any edits.

## Hierarchy

As a player, I want to navigate to different points in the hierarchy.

As a user, I want to have a hierarchy.

As a DM, I upload graphics, and organize them in a hierarchy.

As a user, I want to be able to select element in the hierarchy.

As a user, I want to move elements in the hierarchy.

As a user, I want to delete elements from the hierarchy (with a prompt).

As a user, I want to be able to fold away the hierarchy.

As a user, I want it to work on mobile.

As a user, I want to be able to create labels for hierarchy elements.

As a user, I want hierarchy elements to fold.

## Graphics

As a player, I want to be able to pan and zoom.

As a player, I want each node in the hierarchy to have an associated graphic.

As a player, I want to add annotations to graphics.

As a player, I want my annotations to take the form of ngons.

As a player, I want to edit polygons, and descriptions.

As a player, I want rich text support.

# Developer Use Cases

As a developer, I want to use this for the hierarchy [React Sortable Tree](https://github.com/frontend-collective/react-sortable-tree)

As a developer, I want rich text support [Rich Text Support](https://ourcodeworld.com/articles/read/1065/top-15-best-rich-text-editor-components-wysiwyg-for-reactjs)

As a developer, I want to use the file system for storage.

As a developer, I want a deployment script.
